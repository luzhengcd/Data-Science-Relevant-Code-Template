So the feature engineering in this section would primarily focus on addressing some key points we found in the data exploration part.

The difference between l1 and l2 is that l1 forces some coefficients to be zero to further eliminate insignificant features, while l2 just shrinks the coefficient.
The plot above tells us with larger C comes better auc regardless of penalty term. This is expected as C is the inverse of penalty strength, and larger C means weaker regularization, therefore more features will be selected in the model in the case of l1 and more features will tend to contribute to the classification in the case of l2.

This also explains why after parameter tuning, we do not see significant improvement. The regularization in logistic regression is essentially manipulating the number of features. Decreasing the number of features usually leads to increasing in bais and decreasing in variance. Obviously, our variance is not high even with all features, so fewer features under this scenario increase the bias, causing underfitting.

We have a few parameters to control overfitting in random forest, some are stronger, some are weaker. I will primarily look at the four paramters: min_samples_split, min_samples_leaf, max_features, and max_depth. Among the three parameters, max_depth is the strongest regularizer, so I save it for the last. What's more, instead of using grid search to tune multiple parameters at the same time, I am more inclined to tune them one by one, as it could give me better control on these parameters.

The steps of parameter tuning are first using cross validation on training data with all candidate parameters, and then using the best parameters to train a model on all training data, and test the model on test data.
Let's start with min_samples_split.
